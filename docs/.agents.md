# Agent Guidance (GPT-5.2-Codex)

This file defines how an agent should consume the docs in this folder.

## Scope
- Use [docs/architecture.md](docs/architecture.md), [docs/ui_rules.md](docs/ui_rules.md), and [docs/components.md](docs/components.md) as authoritative summaries.
- Every claim must be traceable to a linked source file. If a claim is not supported by a source link, do not include it.

## Grounding Rules
- No assumptions. If something is not explicitly present in the linked files, say "not observed" or omit it.
- Prefer direct quotes of names (route names, component names, file names) over paraphrase when documenting behavior.
- When unclear, read the referenced file and verify.

## Output Expectations
- Keep architecture, UI rules, and component references aligned with the current code.
- If code changes, update the docs before generating downstream specs.
